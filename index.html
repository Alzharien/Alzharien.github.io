在电脑上配置多个git账号以及hexo

在大部分情况下我们电脑上都是一个账号，配置方面要相对简单很多，如果需要配置两个或以上的git账号的话，SSH就需要额外的配置用来保证把这些SSH区分开来。

1.1首先要设置git的user.name和user.email，在gitbash上使用以下两个指令
‘’’bash
$ git config –global user.name “名字”
$ git config –global user.email “邮箱”
‘’’
没有报错就是登陆成功

1.2如果不想来回切换的话也可以通过设置局部变量的方式解决
‘’’bash
git config –global –unset ‘user.name’
git config –global –unset ‘user.email’
‘’’
删除全局变量设置
然后进入项目的根目录单独设置
‘’’bash
git config user.name xxx
git config user.email xxx@xxx.com
‘’’
在设置完以后，如果需要部署git，在git init之后一定要再次检查设置，如果设置不对需要重新设置

1.3确认没有报错以后生成SSH
‘’’bash
$ ssh-keygen -t rsa -C “user@email.com“
‘’’
接下来会问你存储SSH的地址，如果以前没存过直接按三次回车就行，如果以前有的话需要设置一个新的名字。

接下来查看自己的SSH根目录下有没有配置config文件夹，如果没有则使用以下指令
‘’’bash
$ touch config
‘’’
生成一个config文件夹，进入config文件夹手动配置如下
‘’’bash
Host username1 #后面后缀名可以随意写
HostName github.com
User git #前缀名写对应网站的前缀名
IdentityFile ~/.ssh/id_rsa #对应自己的SSH密匙存储路径

Host username2
HostName github.com #注意缩进
User git
IdentityFile ~/.ssh/id_rsa2 #对应自己另一个SSH密匙存储路径
‘’’
配置完成后将密匙用记事本形式打开，复制里面的内容，进入github账号，在进入Setting，进入SSH and GPG ksys，选择New SSH key，将复制内容黏贴进入内容，title随便填，保存

添加新的SSH密匙到SSH agent中
进入.ssh根目录下，
‘’’bash
$ ssh-add -D #清空缓存
$ ssh-add id_rsa #依次输入密匙名称
$ ssh-add -d_rsa2
‘’’
如果以上命令出现错误，则需要先执行
‘’’bash
$ ssh-agent bash
‘’’
再继续执行

验证配置是否成功
‘’’bash
ssh -T git@github.com #在@的前面写User对应内容，后面写Host的对应内容
ssh -T git@xxxxxxx.github.com
‘’’
以下结果说明SSH正常连接
‘’’bash
Hi ‘用户名’! you ‘ve successfully authenticated, but GitHub does not provide s
hell access
‘’’

部署hexo的部分请参考官方教程，这里仅需要注意关键部分，在部署完以后，hexo文件根目录里的_config.yml文件夹的配置，其中找到deploy部分
‘’’bash
deploy:
type: ‘git’
repo: git@github.com:username/username.github.io.git
brunch: main
‘’’
repo的配置需要采用github账户里面的SSH网址的格式，其中前缀名和后缀名改成在.ssh/config里面配置的内容，brunch请以官方文档为主，按需求自行调整

注：按照此方法设置后，有时候在图片渲染方面，第二个项目有时会出现渲染失败的情况，本人并不知晓原因，欢迎留言讨论。
